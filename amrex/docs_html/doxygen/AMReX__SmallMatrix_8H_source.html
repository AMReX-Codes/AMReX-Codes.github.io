421 /*
422  * Notes on why SmallMatrix matrix{} is zero initialized.
423  *
424  * SmallMatrix is not an aggregate, because it has a user declared default
425  * constructor. The rule is that, for `SmallMatrix matrix{}` with an empty
426  * brace-enclosed initializer list, value-initialization is performed. The
427  * effects of value-initialization of SmallMatrix (which has a user-declared
428  * but not user-provided default constructor) are that the matrix object is
429  * first zero-initialized and then the object's default constructor is
430  * applied. Since the default constructor does nothing, the final result is
431  * the object is zero-initialized.
432  *
433  * Why is SmallMatrix's default constructor user-declared not user-provided?
434  * It's because we first declare it with `SmallMatrix () = default`.
435  *
436  * Reference:
437  * https://en.cppreference.com/w/cpp/language/list_initialization
438  * https://en.cppreference.com/w/cpp/language/value_initialization
439  * https://en.cppreference.com/w/cpp/language/zero_initialization
440  */
