57 /*
58  A Collection of Fortran Array-like Objects
59 
60 
61  The FabArray<FAB> class implements a collection (stored as an array) of
62  Fortran array-like objects. The parameterized type FAB is intended to be
63  any class derived from BaseFab<T>. For example, FAB may be a BaseFab of
64  integers, so we could write:
65 
66  FabArray<BaseFab<int> > int_fabs;
67 
68  Then int_fabs is a FabArray that can hold a collection of BaseFab<int>
69  objects.
70 
71  FabArray is not just a general container class for Fortran arrays. It is
72  intended to hold "grid" data for use in finite difference calculations in
73  which the data is defined on a union of (usually disjoint) rectangular
74  regions embedded in a uniform index space. This region, called the valid
75  region, is represented by a BoxArray. For the purposes of this discussion,
76  the Kth Box in the BoxArray represents the interior region of the Kth grid.
77 
78  Since the intent is to be used with finite difference calculations a
79  FabArray also includes the notion of a boundary region for each grid. The
80  boundary region is specified by the ngrow parameter which tells the FabArray
81  to allocate each FAB to be ngrow cells larger in all directions than the
82  underlying Box. The larger region covered by the union of all the FABs is
83  called the region of definition. The underlying notion is that the valid
84  region contains the grid interior data and the region of definition includes
85  the interior region plus the boundary areas.
86 
87  Operations are available to copy data from the valid regions into these
88  boundary areas where the two overlap. The number of components, that is,
89  the number of values that can be stored in each cell of a FAB, is either
90  given as an argument to the constructor or is inherent in the definition of
91  the underlying FAB. Each FAB in the FabArray will have the same number of
92  components.
93 
94  In summary, a FabArray is an array of FABs. The Kth element contains a FAB
95  that holds the data for the Kth grid, a Box that defines the valid region
96  of the Kth grid.
97 
98  A typical use for a FabArray would be to hold the solution vector or
99  right-hand-side when solving a linear system of equations on a union of
100  rectangular grids. The copy operations would be used to copy data from the
101  valid regions of neighboring grids into the boundary regions after each
102  relaxation step of the iterative method. If a multigrid method is used, a
103  FabArray could be used to hold the data at each level in the multigrid
104  hierarchy.
105 
106  This class is a concrete class not a polymorphic one.
107 
108  This class does NOT provide a copy constructor or assignment operator.
109 */
