19  internal consistency of AMReX, that may also be useful in user
20  code. The argument EX is a C++ expression that you expect to
21  always evaluate to true.
22 
23  When code is compiled with the DEBUG macro set to TRUE, the
24  AMREX_ASSERT(EX) macro expands to a boolean test of the expression EX.
25  If that expression evaluates to true, nothing happens, but if the

27  file and line number of the AMREX_ASSERT(EX) statement, as well as the
